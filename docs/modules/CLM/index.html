    WP05-CLM

    General Description

    The WP05-CLM module is responsible for enabling the multilingual conceptual understanding, annotation, and enrichment of textual data across the DS2 ecosystem. This includes the creation and lifecycle management of linguistic assets such as tokenizers, morphological analyzers, part-of-speech taggers, and annotated corpora in Romanian, Slovenian, and Greek. These resources feed into downstream modules such as SNER, chatbots, and catalogues, supporting both semantic discovery and interaction across dataspaces.

    The CLM module orchestrates the domain-specific generation and refinement of ontologies, ensuring that extracted concepts align with DS2’s cross-linguistic and cross-domain architecture. It also exposes APIs and automated R&D workflows to allow other modules and end-users to register, share, and re-use conceptual resources. By M36, the module reaches full maturity across all supported languages and integrates tightly with the conversational, cataloguing, and lifecycle components of DS2, supporting both in-Dataspace and inter-Dataspace interoperability.

    Subcomponents and Tasks

    The subcomponents of WP05-CLM (Concept Lifecycle Management) are as follows:

    Multilingual NLP Pipeline

    This subcomponent encompasses the development of natural language processing tools for Romanian, Slovenian, and Greek. It includes dedicated tokenizers, morphological analyzers, and part-of-speech (POS) taggers. These linguistic resources are foundational for enabling structured data extraction, annotation, and semantic enrichment across multiple languages within the DS2 dataspace.

    Annotation Framework

    The Annotation Framework facilitates the manual and semi-automated annotation of multilingual texts, enabling entity recognition and concept tagging for downstream semantic modules. This subcomponent is integrated with language-specific models and serves as the core data provider for SNER modules.

    Ontology Generation Engine

    Responsible for the creation of domain-specific ontologies based on extracted concepts from annotated data. It leverages structured NLP outputs to identify hierarchical relationships, synonyms, and concept mappings. The engine supports both initial ontology bootstrapping and iterative enrichment cycles, ensuring alignment with DS2’s modular and federated structure.

    Integration Layer (Chatbot & Catalogue)

    This subcomponent ensures that all outputs from the CLM module (e.g., ontologies, annotated corpora, structured entities) are properly integrated into the DS2 Chatbot and Concept Catalogue modules. It exposes APIs and adapters that translate internal formats into standardized metadata schemas consumable by other DS2 components.

    API Gateway

    Provides RESTful endpoints that allow external modules and tools to access CLM-generated assets, such as ontologies, tagged datasets, and linguistic models. The gateway supports secured and versioned access and facilitates communication with other WP modules, including registration and configuration interfaces.

    Automated Ontology R&D

    This subcomponent supports research and development workflows for automated ontology learning, including the use of statistical, rule-based, and AI-driven techniques. It enables the experimentation and refinement of concept extraction and relation modeling, feeding results back into the Ontology Generation Engine.

    Registration & Compliance Processor

    Responsible for validating, registering, and publishing CLM outputs (e.g., ontologies, annotation sets, models) in accordance with DS2 governance rules. It interacts with the DS2 metadata registry and ensures that all outputs are properly described, versioned, and traceable across the dataspace lifecycle.

    Installation

    ⚠️ This module is currently integrated through the central IDT system (in progress).
    For now, manual deployment is not supported. Documentation will be updated once IDT-based integration is finalized.
